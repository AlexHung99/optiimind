Backend & Runtime Integration
後端與執行期整合

Keep the UI stable while choosing the auth stack that fits your game.

保留同一套 UI，自由選擇適合遊戲的驗證架構。

The shared backend base class wires UI events to validation, async auth operations, status feedback, session persistence, and post-login UnityEvents. Concrete backends only implement the service-specific calls.

共用後端基底類別會把 UI 事件接到驗證、非同步登入流程、狀態回饋、Session 保存與登入後 UnityEvent；具體後端只需實作各服務的呼叫。

  • Self-hosted REST API自架 REST APIConfigurable endpoints for login, register, reset, refresh, social login, guest sign-in, account upgrade, logout, and account deletion.可設定登入、註冊、重設、刷新、社群登入、訪客登入、帳號升級、登出與帳號刪除端點。
  • Unity Gaming ServicesUnity Gaming ServicesDrop-in component for UGS Authentication projects using username/password flows.提供可直接掛載的 UGS Authentication 元件，支援帳號密碼流程。
  • Firebase AuthenticationFirebase AuthenticationEmail/password Firebase backend with reset email support when the Firebase SDK and define are enabled.啟用 Firebase SDK 與 define 後，可使用 email/password 後端與重設密碼 email。
  • Social Login Bridge社群登入橋接Google, Facebook, LINE, and Apple buttons appear automatically when provider components are added.加入供應商元件後，Google、Facebook、LINE 與 Apple 按鈕會自動顯示。

1. 套件簡介與核心優勢

OptiLiteDB 是一款專為 Unity 開發者打造的本機 SQLite 資料庫整合方案。它將「視覺化編輯器資料庫瀏覽器」與「輕量極速的執行時 C# API」完美融合，免除跨平台編譯衝突，提供開發與運行時的最佳效能。

💡 原生動態連結 (Zero-DLL) 技術
系統會自動、安全地在執行時加載各大平台內建的 SQLite 原生庫（Windows 10+ 的 winsqlite3，以及 macOS/iOS/Linux 的系統 libsqlite3），您無需隨套件打包任何二進位 DLL 檔案，包體更小、極致安全。

2. 編輯器視覺化工具 (DB Browser) 操作說明

開啟資料庫瀏覽器

  • 方式一： 點選頂部選單 Window ▸ OptiLiteDB ▸ Database Browser（或 Tools ▸ OptiLiteDB）。
  • 方式二（推薦、最快）： 在 Project 視窗中，直接雙擊 (Double-Click) 任何 SQLite 檔案（支援 .db, .sqlite, .sqlite3, .db3 格式），或右鍵選單點選 Open in OptiLiteDB Browser。

核心介面與操作

  • 資料庫切換下拉選單： 自動掃描並列出 Assets 底下所有的 SQLite 檔案，一鍵輕鬆切換。
  • Read-only 模式勾選： 支援以唯讀形式開啟資料庫，防止在瀏覽測試資料時不小心更動數據。
  • 直覺式資料編輯（Browse Data 分頁）：
    • 雙擊儲存格 (Cell)： 直接輸入內容並按 Enter 或點選別處，資料便會即時且自動同步至 SQLite 實體資料庫。
    • 快速設為 NULL： 選取儲存格後點選工具列 Set NULL 按鈕，或在儲存格上點擊滑鼠右鍵選擇 Set NULL。
    • 點選表頭排序： 點選任何欄位標題，資料將立即在資料庫端執行 ORDER BY 進行升冪或降冪排序（支援跨分頁精確排序）。
    • 即時資料篩選 (Filter)： 支援輸入標準 SQL WHERE 條件式（例如：qty > 5 AND name LIKE 'S%'）來過濾網格資料。
    • 資料列增刪： 點選 + Add Row 快速插入一列（套用預設值）；選取多列後點選 - Delete Selected 進行安全刪除。
  • 自訂 SQL 執行（Execute SQL 分頁）：
    • 在文字框內輸入任何自訂 SQL 語法，按下 Ctrl + Enter（macOS 為 Command + Enter）即可執行。
    • SELECT 語法會即時在下方渲染出高解析度的數據網格；INSERT/UPDATE/DELETE 則顯示受影響列數。支援同時執行多條 ; 隔開的語句。

3. 進階 CSV 匯出與匯入

匯出 CSV

在 Browse Data 分頁，點選 Export CSV ➔ 選擇分隔符號（逗號/分號/Tab/直線）➔ 儲存即可，數據（含編碼過的 BLOB）均會依 RFC 4180 標準安全跳脫匯出。

匯入 CSV（含事務回滾保護）

點選 Import CSV 並選擇本機 CSV 檔案後，將彈出配置對話框：

  • 設定選定： 自訂分隔符號，並可設定首列是否為欄位標頭。
  • 清空匯入模式： 可勾選 Clear table before importing（在匯入寫入前，先清空該資料表內的所有既有數據）。
  • 欄位映射對應 (Column Mapping)： 視覺化配對資料表與 CSV 的列索引，預設會自動依首列標頭名稱進行比對。
⚠️ 交易保護安全機制 (Rollback)
清空與寫入指令均在同一個資料庫交易（Transaction）中執行。如果 CSV 在匯入途中發生任何格式錯誤，整筆操作會自動回滾（Rollback），您資料表原本的歷史資料將完好無損，避免發生資料表被清空、寫入卻失敗的窘境。

4. 視覺化建表與結構修改 (Alter Table)

視覺化建表

點選左側 Sidebar 的 + Table，可編輯表名、指定型別（INTEGER、TEXT、REAL、BLOB、NUMERIC）、勾選限制（主鍵 PK、自增 AI、非空 NN、唯一 U），並可使用 ▲ / ▼ 按鈕任意調整欄位排列順序。

無損重建式修改 (Alter Table)

選定資料表並點擊 Edit。在這裡你可以隨意重命名欄位、變更型別、增減或重排欄位。當點選套用時：

  1. 系統會在背景建立新表，並透過交易將原表同名欄位的數據安全拷貝過去，然後重命名。
  2. 自動重建索引與觸發器： 系統會提前擷取原表上綁定的自訂 Index 與 Trigger，並在重建後自動將其重新建立套用（會自動改寫 DDL 中的關聯表名），保障資料庫完整性。

5. 執行時 C# API 開發指南

在遊戲執行時，你可以在 C# 腳本中無縫讀寫 SQLite 資料庫（完全相容於打包發布）：

using UnityEngine;
using OptiLiteDB;

public class SampleSave : MonoBehaviour
{
    void Start()
    {
        // 建議在 Runtime 將資料庫放在 Application.persistentDataPath 中
        string path = System.IO.Path.Combine(Application.persistentDataPath, "save.db");
        
        using (var db = OptiLite.Open(path, readOnly: false, create: true))
        {
            // 1. 建立資料表
            db.Execute("CREATE TABLE IF NOT EXISTS player (id INTEGER PRIMARY KEY, name TEXT, hp INTEGER);");
            
            // 2. 參數化安全寫入 (防 SQL 注入)
            db.Execute("INSERT INTO player (name, hp) VALUES (?, ?);", "Hero_Alex", 100);
            
            // 3. 讀取查詢資料
            OptiLiteResult res = db.Query("SELECT name, hp FROM player ORDER BY id DESC LIMIT 1;");
            if (res.HasRows)
            {
                Debug.Log($"Loaded: {res.Rows[0][0]} with HP {res.Rows[0][1]}");
            }
        }
    }
}

6. AI Agent / 自動化專用 API

OptiLiteDB 內建了專為 AI 助理與自動化腳本設計的單一進入點靜態類別 OptiLiteEditorAPI。它會在內部自動進行「路徑轉換」、「自動 Disposing 連線防止檔案鎖死」以及「編輯器 UI 即時刷新重繪」：

using OptiLiteDB.Editor;

// AI 專用：一列指令完成寫入，並會使開著的 DB Browser 視窗瞬間自動刷新資料
OptiLiteEditorAPI.Execute(
    "Assets/game_data.db", 
    "INSERT INTO items (name, price) VALUES (?, ?);", 
    "Master Sword", 9999.0
);

// AI 專用：一列指令讀取資料
OptiLiteResult res = OptiLiteEditorAPI.Query("Assets/game_data.db", "SELECT * FROM items;");

1. Package Introduction & Core Advantages

OptiLiteDB is a lightweight, high-performance SQLite integration for Unity developers. It seamlessly merges an in-editor visual DB Browser with an ultra-fast runtime C# API, fully compatible with Unity 2022.3 up to Unity 6 (6000.x).

💡 Native Dynamic Linking (Zero-DLL)
It dynamically binds to native SQLite libraries pre-installed in the host operating system (winsqlite3 on Windows, and system libsqlite3 on macOS/iOS/Linux). There are no DLL binaries to manage, reducing build footprint and avoiding cross-platform compile errors.

2. In-Editor Visual DB Browser Guide

How to Open the DB Browser

  • Method 1: Select the menu item Window ▸ OptiLiteDB ▸ Database Browser (or Tools ▸ OptiLiteDB ▸ Database Browser).
  • Method 2 (Recommended & Fastest): Simply double-click any SQLite database file (supports .db, .sqlite, .sqlite3, .db3) in your Project pane, or right-click the asset and choose Open in OptiLiteDB Browser.

Core Interface & Interactions

  • Database Selector Dropdown: Automatically scans and indexes all SQLite files under your Assets folder for seamless hot-swapping.
  • Read-only Protection Mode: Toggle Read-only mode to securely browse game balances or design-sheets without accidental modifications.
  • Intuitive Cell Editing (Browse Data Tab):
    • Double-click cell: Edit cell values in place. Press Enter or unfocus to commit modifications directly to the physical database.
    • Set to NULL: Click the Set NULL button in the toolbar, or simply right-click a cell and choose Set NULL.
    • Header-click Column Sorting: Click column headers to toggle ascending/descending order instantly (executes database-side ORDER BY).
    • Live Data Filtering (Filter): Apply standard SQL WHERE expressions (e.g., qty > 5 AND name LIKE 'S%') to instantly filter rows.
    • Insert/Delete: Click + Add Row to append a default row; select rows and click - Delete Selected to safely remove entries.
  • Custom SQL Panel (Execute SQL Tab):
    • Enter arbitrary SQL statements and execute them instantly with the Ctrl + Enter (macOS: Command + Enter) shortcut.
    • SELECT queries automatically render high-resolution result grids below, while INSERT/UPDATE/DELETE queries display the count of rows affected. Multiple statements separated by ; are supported.

3. Advanced CSV Import & Export

Export CSV

Under the Browse Data tab, click Export CSV ➔ Select field delimiter (Comma / Semicolon / Tab / Pipe) ➔ Save. Output (including Base64 encoded BLOBs) is safely quoted conforming to RFC 4180.

Import CSV (with Transaction Protection)

Click Import CSV and select a file to open the Configuration Dialog:

  • Delimiter & Header Toggle: Set your CSV delimiter and toggle whether the first row represents the column headers.
  • Clear Table Mode: Enable Clear table before importing to truncate the table prior to inserting.
  • Visual Column Mapping: Assign each destination table column to any index in your CSV, with automatic matches calculated based on header names.
⚠️ Transaction Rollback Protection
The deletion and insertion statements execute inside a single database transaction. If the CSV parsing fails midway, the entire operation is automatically rolled back (Rollback) — your original table data is preserved untouched.

4. Schema & Table Designer

Create Table Visually

Click + Table on the sidebar. Edit the table name, add columns, select data types (INTEGER, TEXT, REAL, BLOB, NUMERIC), assign constraints (PK, AutoIncrement, Not Null, Unique), and reorder columns dynamically using the ▲ / ▼ buttons.

Lossless Reconstructive Table Editing (Alter Table)

Select a table and click Edit. You can visually rename, change types, add, remove, or reorder columns. Upon clicking apply:

  1. OptiLiteDB runs a transaction-backed rebuild that creates a safe temporary table, copies existing data over matching column names, drops the old table, and renames the temporary table.
  2. Auto Index & Trigger Reconstruction: User-defined indexes and triggers are captured beforehand, and re-created on the modified table automatically (rewriting references).

5. Runtime C# API Development Guide

Read and write SQLite databases seamlessly in your game scripts (perfect for built standalone players):

using UnityEngine;
using OptiLiteDB;

public class SampleSave : MonoBehaviour
{
    void Start()
    {
        // Use Application.persistentDataPath for local saves in runtime builds
        string path = System.IO.Path.Combine(Application.persistentDataPath, "save.db");
        
        using (var db = OptiLite.Open(path, readOnly: false, create: true))
        {
            // 1. Create table
            db.Execute("CREATE TABLE IF NOT EXISTS player (id INTEGER PRIMARY KEY, name TEXT, hp INTEGER);");
            
            // 2. Safe Parametric Insertions (Protects against SQL Injection)
            db.Execute("INSERT INTO player (name, hp) VALUES (?, ?);", "Hero_Alex", 100);
            
            // 3. Query records
            OptiLiteResult res = db.Query("SELECT name, hp FROM player ORDER BY id DESC LIMIT 1;");
            if (res.HasRows)
            {
                Debug.Log($"Loaded: {res.Rows[0][0]} with HP {res.Rows[0][1]}");
            }
        }
    }
}

6. AI Agent & Automation API

OptiLiteDB features OptiLiteEditorAPI, a high-level static facade designed specifically for AI Agents and custom editor scripting. It automatically handles absolute path conversions, connections auto-disposal, and forces database asset updates:

using OptiLiteDB.Editor;

// AI Special: single-line insertion which instantly repaints any opened DB Browser window
OptiLiteEditorAPI.Execute(
    "Assets/game_data.db", 
    "INSERT INTO items (name, price) VALUES (?, ?);", 
    "Master Sword", 9999.0
);

// AI Special: single-line query
OptiLiteResult res = OptiLiteEditorAPI.Query("Assets/game_data.db", "SELECT * FROM items;");
